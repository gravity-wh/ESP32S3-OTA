通讯协议说明

本产品通讯接口为标准RS485接口，通信格式:115200 n 8 1传感器为从机，传感器地址:01(0x01)默认;

MODBUS-RTU模式符合GB/T19582.3-2008中的相关定义

MODBUS-RTU协议数据帧结构

| 地址码  | 功能码  | 数据区  | CRC校验 |
| ------- | ------- | ------- | ------- |
| 1 Bytes | 1 Bytes | N Bytes | 2 Bytes |

注:
|地址码:1个字节的从机地址码，默认为0x01;
| 功能码:1字节的功能码;
|数据区:N字节，包含起始地址、数据等;
ICRC校验:2字节，小端模式，校验数据的范围为:地址码+功能码+数据区

MODBUS-RTU协议涉及的功能码

| 功能码 | 特性 | 作用               |
| ------ | :--: | ------------------ |
| 0x03   |  读  | 读取从机内指定信息 |
| 0x06   |  写  | 对从机指定位置写入 |

MODBUS-RTU模式中功能寄存器地址配置表

| 地址   | 特性  | 功能     | 说明                                                                               |
| ------ | ----- | -------- | ---------------------------------------------------------------------------------- |
| 0x0010 | 读    | 距离     |                                                                                    |
| 0x0015 | 读/写 | 从机地址 | 默认地址为0x01，对未知的从机设备可以用0xFF进行代替，可设置地址范围为0x0001一0x00FE |

协议举例
例1:读取地址为0x0001设备的距离值;

[20:57:20.824]发送一01 03 00 10 00 01 85 CF

[20:57:20.888]接收-0103 02 01 12 39 D9

解析主机发送的指令
|地址码 | 功能码|数据区| CRC校验|功能|
| 0x01|0x03 | 0x0010 | 0x0001 | 0x85CF|读取地址0x0001的机器距离|
注:
1、地址码0x01，需要进行操作的目标设备地址
2、功能码0x03，进行读取操作
3、数据区地址码0x0010;查询功能寄存器地址配置表可知距离数据存在0x0010
4、数据区数据码0x0001;同注3可知仅读距离时为0x0001

[20:57:20.888]接收-01 03 02 01 12 39 D9
1、发送数据的从机地址为0x01;
2、功能码0x03，此帧为读取数据;
3、数据长度为0x02，数据长度为2字节;
4、数据为0x01、0x12;数据合并为0x0112，换算成十进制为274，即此时通道1测量的距离值为274mm。图示:

| 地址码 | 功能码 | 数据区-地址 | 数据   | CRC校验 | 功能                               | 备注 |
| :----: | ------ | :---------: | ------ | ------- | ---------------------------------- | ---- |
|  0x01  | 0x06   |   0x0015   | 0x0002 | 0x19CF  | 将地址0x0001的机器修改为地址0x0002 |      |
|  0x02  | 0x03   |   0x0010   | 0x0001 | 0x85FC  | 读取地址0x0002的机器通道1距离      |      |
|  0x02  | 0x06   |   0x0015   | 0x0001 | 0x59FD  | 将地址0x0002的机器修改为地址0x0001 |      |
|  0xFF  | 0x06   |   0x0015   | 0x0001 | 0x4c10  | 将未知地址的机器修改为地址0x0001   |      |
