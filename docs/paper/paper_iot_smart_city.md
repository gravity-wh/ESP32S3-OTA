# 面向智慧城市的物联网系统开发

**作者：Andrea Zanella, Senior Member, IEEE, Nicola Bui, Angelo Castellani,
Lorenzo Vangelista, Senior Member, IEEE, and Michele Zorzi, Fellow, IEEE**

---

### **【摘要】**

物联⽹（IoT）能够透明、无缝地整合大量不同且异构的终端系统，同时为开发大量数字服务提供对选定数据子集的开放访问。因此，构建通用的物联⽹架构是一项非常复杂的任务，这主要是因为此类系统中可能涉及的设备、链路层技术和服务种类繁多。本文重点关注城市物联⽹系统，尽管城市物联⽹本身仍是一个相当广泛的类别，但其应用领域却具有鲜明的特点。事实上，城市物联⽹旨在支持智慧城市愿景，该愿景旨在利用最先进的通信技术，为城市管理和市民提供增值服务。因此，本文对城市物联⽹的使能技术、协议和架构进行了全面的调研。此外，本文还将介绍和讨论帕多瓦智慧城市项目中采用的技术解决方案和最佳实践指南。该项目是与意大利帕多瓦市政府合作，在帕多瓦市部署的物联⽹岛的概念验证项目。

**关键词**：受限应用协议  (CoAP)；高效  XML  交换  (EXI)；⽹络架构；传感器系统集成；服务功能和管理

### **一、引言**

物联⽹  (IoT)  是一种新兴的通信范式，它设想在不久的将来，日常生活中的物品将配备微控制器、数字通信收发器和相应的协议栈，使它们能够彼此通信并与用户通信，从而成为互联⽹不可或缺的一部分  [1]。因此，物联⽹的概念旨在使互联⽹更加沉浸式和普及。
此外，它还使用户能够轻松访问和交互各种设备，例如家用电器、监控摄像头、监控设备等。在这种复杂的背景下，将物联⽹范式应用于城市环境尤为引人关注，因为它响应了许多国家政府大力推动在公共事务管理中采用信息通信技术解决方案的号召，从而实现所谓的智慧城市理念[4]。尽管目前尚未有正式且被广泛接受的“智慧城市”定义，但其最终目标是更有效地利用公共资源，提高为市民提供的服务质量，同时降低公共管理部门的运营成本。实现这一目标可以通过部署城市物联⽹来实现，即构建一个通信基础设施，为市民提供统一、便捷且经济的公共服务访问渠道，从而释放潜在的协同效应，并提高公共服务的透明度。城市物联⽹确实可以为传统公共服务的管理和优化带来诸多益处，例如交通和停⻋、照明、公共区域的监控和维护、文化遗产保护、垃圾收集、医院和学校的卫生状况等。此外，城市物联⽹收集的各种类型数据也可用于提高地方政府的透明度，促进其对市民的行动，增强市民对城市状况的了解，鼓励市民积极参与公共管理，并促进基于物联⽹服务的新服务的开发。[5]  因此，将物联⽹范式应用于智慧城市对地方和区域政府尤其具有吸引力，这些政府可能会成为此类技术的早期采用者，从而起到催化剂的作用，推动物联⽹范式在更大范围内的采用。

本文旨在探讨城市物联⽹设计的一般参考框架。我们首先描述了城市物联⽹的具体特征，以及可能推动地方政府采用城市物联⽹的服务。然后，我们概述了基于⽹络的物联⽹服务设计方法及其相关协议和技术，并探讨了它们在智慧城市环境中的适用性。最后，我们以“帕多瓦智慧城市”项目为例，阐述了我们的讨论。该项目是在意大利帕多瓦市部署物联⽹岛的概念验证，该物联⽹岛与市政数据⽹络互联。在此，我们描述了为实现该物联⽹岛而采用的技术方案，并报告了系统在最初运行阶段收集的一些测量数据。

### 二、智慧城市的概念和服务

根据派克研究公司（Pike  Research）发布的《智慧城市》报告，到2020年，智慧城市市场规模预计将达到数千亿美元，年支出将接近160亿美元。这一市场源于关键行业和科技的协同互联。

服务领域，例如智慧治理、智慧交通、智慧公用事业、智慧建筑和智慧环境，也属于智慧城市范畴。欧洲智慧城市项目（http://www.smart‑cities.eu）也考虑了这些领域，旨在制定一套排名标准，用于评估欧洲城市的“智慧化”水平。然而，由于诸多政治、技术和金融方面的障碍[6]，智慧城市市场尚未真正发展起来。

在技术方面，最相关的问题在于目前城市发展中使用的异构技术缺乏互操作性。在这方面，物联⽹愿景可以成为构建统一的城市级信息通信技术平台的基础，从而释放智慧城市愿景的潜力[8]，[9]。

最后，就财务层面而言，尽管近期已有一些举措试图弥补这一空白[10]，但目前仍缺乏清晰的商业模式。全球经济形势不利，导致公共服务投资普遍缩减，使情况雪上加霜。这种情况阻碍了潜力巨大的智慧城市市场成为现实。摆脱困境的一个可能途径是，首先开发那些兼具社会效益和明确投资回报的服务，例如智能停⻋和智能建筑，从而带动其他增值服务的发展[10]。

在本节的其余部分，我们将概述一些可能由城市物联⽹范式实现的服务，这些服务在智慧城市背景下具有潜在的应用价值，因为它们能够实现双赢：既能提高并增强为市民提供的服务质量，又能通过降低运营成本为城市管理部门带来经济效益[6]。为了更好地了解这些服务所需技术的成熟度，我们在表  I  中对这些服务进行了概述。

### 三、城市物联⽹建筑

从第二节中描述的服务分析可以看出，大多数智慧城市服务都基于集中式架构，其中部署在城市区域内的密集且异构的外围设备生成不同类型的数据，然后通过合适的通信技术将这些数据传递到控制中心，在那里进行数据存储和处理。

因此，城市物联⽹基础设施的一个主要特征是能够将不同的技术与现有的通信基础设施相集成，从而支持物联⽹的逐步演进，实现其他设备的互联互通以及新功能和服务的实现。另一个基本方面是必须确保政府部门和市民能够轻松访问城市物联⽹收集的部分数据，以提高政府部门对城市问题的响应速度，并促进市民对公共事务的认识和参与[9]。

在本节的其余部分，我们将描述城市物联⽹系统的不同组成部分，如图  1  所示。我们首先描述用于设计物联⽹服务的  Web  服务方法，这需要在⽹络的不同元素中部署合适的协议层，如协议栈所示。

#### A.面向物联⽹服务架构的Web服务方法

尽管在物联⽹领域，许多不同的标准仍在努力成为参考标准和最广泛采用的标准，但在本节中，我们将特别关注  IETF  标准，因为它们是开放的、免版税的，基于互联⽹最佳实践，并且可以依靠广泛的社区。

IETF  的物联⽹标准采用了一种用于物联⽹服务的  Web  服务架构，该架构已被文献广泛证实是一种极具前景且灵活的方法。事实上，Web  服务允许实现一个灵活且可互操作的系统，该系统可以通过采用称为表述性状态转移  (ReST)  [18]  的基于  Web  的范式扩展到物联⽹节点。根据  ReST  范式设计的物联⽹服务与传统  Web  服务具有高度相似性，从而极大地促进了最终用户和服务开发人员对物联⽹的采用和使用。他们可以轻松地重用从传统  Web  技术中获得的大量知识，用于开发包含智能对象的⽹络服务。Web  服务方法也得到了  IETF、ETSI  和W3C  等国际标准化组织以及  SENSEI、  IoT‑A和  SmartSantander等欧洲物联⽹研究项目的推广。

图  2  展示了城市物联⽹系统的参考协议架构，其中包含一个非约束协议栈和一个约束协议栈。非约束协议栈由目前互联⽹通信的事实标准协议组成，这些协议通常被常规互联⽹主机使用，例如  XML。

##### 1）数据格式：

如前所述，城市物联⽹范式对数据可访问性提出了特定要求。在基于Web服务的架构中，数据交换通常伴随着对传输内容的描述，这种描述通常使用语义表示语言，其中可扩展标记语言（XML）可能是最常用的。然而，XML消息的大小通常对于典型物联⽹设备的有限容量而言过大。此外，XML表示的文本特性使得CPU受限的设备解析消息比二进制格式更加复杂。鉴于这些原因，万维⽹联盟（W3C）工作组提出了EXI格式[19]，该格式使得即使是资源非常有限的设备也能原生支持和生成使用与XML兼容的开放数据格式的
消息。

##### 2）应用层和传输层：

如今，互联⽹上的大部分流量都是通过基于  TCP  的HTTP  协议在应用层传输的。然而，原生  HTTP  协议冗长且复杂，使其不适合
直接部署在资源受限的物联⽹设备上。事实上，在物联⽹环境中，HTTP  协议易于阅读的格式（这曾是其在传统⽹络中取得成功的原因之一）反而会成为一种限制因素，因为物联⽹需要处理大量高度相关（因而也存在冗余）的数据。此外，HTTP  通常依赖于  TCP  传输协议，而  TCP  在资源受限的设备上扩展性不佳，在丢包环境下处理小流量数据时性能会很差。

##### 3）网络层：

IPv4  是互联⽹主机支持的主要寻址技术。然而，IANA（国际互联⽹地址分配机构）指出，IPv4  仍然是互联⽹领域领先的寻址技术。IP地址分配机构近日宣布IPv4地址块已耗尽。物联⽹⽹络预计将包含数十亿个节点，每个节点（原则上）都应具有唯一的地址。IPv6标准[24]为此问题提供了一种解决方案，它提供了一个128位的地址字段，从而可以为物联⽹⽹络中的任何节点分配唯一的IPv6地址。

#### B.链路层技术

由于城市物联⽹系统本身部署范围广，因此需要一套链路层技术，既能轻松覆盖广阔的地理区域，又能支持由大量小型数据流聚合而成的海量流量。基于这些原因，实现城市物联⽹系统的链路层技术可分为非约束型技术和约束型技术。第一类技术包括所有传统的局域⽹  (LAN)、城域⽹  (MAN)  和广域⽹  (WAN)  通信技术，例如以太⽹、Wi‑Fi、光纤、宽带电力线通信  (PLC)  以及蜂窝技术，如  UMTS  和  LTE。这些技术通常具有高可靠性、低延迟和高传输速率（兆比特/秒或更高）的特点，但由于其固有的复杂性和能耗，通常不适用于外围物联⽹节点。

相反，受限的物理层和链路层技术通常具有低能耗和相对较低的传输速率，通常小于  1Mbit/s。此类技术中较为突出的方案包括  IEEE  802.15.4  [27]、[28]、蓝牙和低功耗蓝牙、IEEE  802.11  低功耗、电力线通信  (PLC)  [29]、近场通信  (NFC)  和射频识别  (RFID)  [30]。这
些链路通常存在较长的延迟，主要原因有两个：

- 1) 物理层固有的低传输速率；
- 2) 节点为节省

能源而实施的节能策略，通常涉及短时工作循环。

#### C.设备

最后，我们描述了实现城市物联⽹所必需的设备，并根据它们在通信流中所处的位置进行分类。

##### 1）后端服务器：

在系统的根部，我们发现后端服务器位于控制中心，数据在这里被收集、存储和处理，以产生增值服务。

原则上，后端服务器并非物联⽹系统正常运行的必要条件，但它们却是城市物联⽹的基本组成部分，能够通过传统⽹络基础设施促进对智慧城市服务和开放数据的访问。通常用于与物联⽹数据源对接的后端系统包括以下⼏种。

- 数据库管理系统：这些系统负责存储物联⽹外围节点（例如传感器）产生的大量信息。根据具体的使用场景，这些系统的负载可能非常大，因此需要对后端系统进行合理的容量规划。
- ⽹站：人们对⽹络界面的广泛熟悉使其成为实现物联⽹系统与“数据消费者”（例如公共机构、服务运营商、公用事业提供商和普通公民）之间互操作的首选方式。
- 企业资源计划系统  (ERP)：ERP  组件支持各种业务功能，是管理复杂组织（例如市政管理机构）信息流的重要工具。将  ERP  组件与收集物联⽹  (IoT)  生成数据的数据库管理系统对接，可以简化对  IoT  收集的海量数据的管理，从而能够根据信息流的性质和相关性对其进行分类，并简化新服务的创建。

##### 2）⽹关：

在物联⽹的“边缘”，我们发现了⽹关，其作用是将终端设备连接到系统的主通信基础设施。参考图  2  所示的概念协议架构，⽹关需要提供非约束协议与其约束对应协议（例如  XML‑EXI、HTTP‑CoAP、IPv4/v6‑6LoWPAN）之间的协议转换和功能映射。

##### 3）物联网外围节点：

最后，在物联⽹系统的外围，我们发现了负责生成数据并将其传输到控制中心的设备，这些设备通常被称为物联⽹外围节点，或简称为物联⽹节点。一般来说，这些设备的成本非常低，从  10  美元甚至更低起，具体取决于板上安装的传感器/执行器的类型和数量。物联⽹节点可以根据多种特性进行分类，例如供电方式、⽹络角⾊（中继或叶节点）、传感器/执行器设备以及支持的链路层技术。

### 四、实验研究：帕多瓦智慧城市

本文讨论的框架已成功应用于物联⽹系统的多个不同用例。例如，帕多瓦大学部署的拥有300多个节点的实验性无线传感器⽹络测试平台[31][32]就是根据这些指导原则设计的，并已成功用于实现智能电⽹[33]和医疗保健[34]服务的概念验证演示。

在本节中，我们将介绍一个名为“帕多瓦智慧城市”的城市物联⽹的实际应用，该应用已在帕多瓦市实现；这要归功于公共和私人各方的合作，例如赞助该项目的帕多瓦市政府、提供项目理论背景和可行性分析的帕多瓦大学信息工程系，以及专门从事创新物联⽹解决方案开发的帕多瓦大学衍生公司  Patavina  Technologies  srl，该公司开发了物联⽹节点和控制软件。

帕多瓦智慧城市的主要目标是促进公共管理部门尽早采用开放数据和信息通信技术解决方案。目标应用是一个环境数据采集系统，该系统通过安装在路灯杆上的无线节点收集环境数据并监测公共路灯照明。这些节点配备各种传感器，并通过⽹关单元连接到互联⽹。该系统能够采集重要的环境参数，例如二氧化碳浓度、空气温度和湿度、振动、噪声等，同时提供一种简单而精确的机制，通过测量光照强度来检查公共照明系统的运行状况。

每个站点的强度各不相同。即使该系统只是物联⽹概念的一个简单应用，它仍然涉及多种不同的设备和链路层技术，因此代表了设计城市物联⽹时需要考虑的大多数关键问题。下文将概述系统中涉及的设备类型及其作用。

受限链路层技术：安装在路灯杆上的物联⽹节点构成了一个基于IEEE  802.15.4受限链路层技术的6LoWPAN多跳云。路由功能由IPv6低功耗有损⽹络路由协议（RPL）[35]提供。物联⽹节点被分配唯一的IPv6地址，并根据6LoWPAN标准进行适当压缩。每个节点都可以单独访问。通过  IPv6/6LoWPAN  从互联⽹上的任何位置访问。各个节点共同将其数据传输到汇聚节点，汇聚节点是外部节点的唯一连接点。或者，每个节点可以通过运行  CoAP  服务器来发布自己的功能和数据，尽管此功能尚未在测试平台中实现。无论采用哪种方式，都需要一个⽹关来将  6LoWPAN  云连接到互联⽹，并执行上一节中描述的所有转码操作。

WSN⽹关：⽹关的作用是将传感器云中使用的受限链路层技术与用于连接中央后端服务器的传统WAN技术对接。因此，⽹关扮演着
6LoWPAN边界路由器和RPL根节点的角⾊。此外，由于传感器节点不支持CoAP服务，⽹关还充当传感器云的汇聚节点，收集所有需要导出到后端服务的数据。与后端服务的连接通过常见的非受限通信技术提供，在本例中为光纤。

这样的系统可以逐步扩展，以包含其他类型的物联⽹节点或物联⽹节点云，前提是每个物联⽹节点都满足特定条件。

### 五、结论

本文分析了目前可用于实施城市物联⽹的解决方案。所讨论的技术已接近标准化，行业参与者已积极投入生产利用这些技术实现相关应用的设备，例如第二节中所述的应用。事实上，尽管物联⽹系统的设计方案选择范围相当广泛，但开放且标准化的协议数量却相对较少。此外，相关技术已达到成熟水平，足以支持物联⽹解决方案和服务的实际应用，首先可开展现场试验，以期消除目前阻碍物联⽹范式大规模应用的诸多不确定性。文中还介绍了一个与意大利帕多瓦市合作部署的具体概念验证项目，该项目是物联⽹范式应用于智慧城市的典型案例。
